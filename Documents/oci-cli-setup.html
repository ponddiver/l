OCI CLI Setup Guide

Complete installation and configuration guide for Oracle Cloud Infrastructure Command Line Interface

Overview

The Oracle Cloud Infrastructure Command Line Interface (OCI CLI) is a powerful tool that allows you to manage your Oracle Cloud Infrastructure resources from the command line. This guide will walk you through the installation and configuration process.

Prerequisites

  • An Oracle Cloud Infrastructure account
  • Python 3.8 or later (Python 3.6+ is supported but 3.8+ is recommended)
  • Administrator or sudo privileges on your machine
  • An active internet connection

Installation

Windows

Option 1: Using PowerShell (Recommended)

powershell -NoProfile -ExecutionPolicy Bypass -Command "iex ((New-Object System.Net.WebClient).DownloadString('https://raw.githubusercontent.com/oracle/oci-cli/master/scripts/install/install.ps1'))"

Option 2: Using Python pip

pip install oci-cli

macOS and Linux

Option 1: Using the installer script (Recommended)

bash -c "$(curl -L https://raw.githubusercontent.com/oracle/oci-cli/master/scripts/install/install.sh)"

Option 2: Using Python pip

pip3 install oci-cli
Note: The installer script automatically handles all dependencies and adds OCI CLI to your PATH. When prompted, accept the default installation directory or specify a custom location.

Verify Installation

After installation, verify that OCI CLI is properly installed:

oci --version

You should see output similar to:

3.45.0

Configuration

Interactive Setup

Run the configuration command to set up your OCI CLI:

oci setup config

You will be prompted to provide the following information:

  1. Config file location: Press Enter to accept the default (~/.oci/config) or specify a custom path
  2. User OCID: Find this in the OCI Console under Identity → Users → User Details
  3. Tenancy OCID: Find this in the OCI Console under Governance → Tenancy Details
  4. Region: Enter your home region (e.g., us-ashburn-1, us-phoenix-1)
  5. Generate API key: Type Y to generate a new API key pair

API Key Setup

If you chose to generate a new API key, the CLI will create:

  • A private key file (typically ~/.oci/oci_api_key.pem)
  • A public key file (typically ~/.oci/oci_api_key_public.pem)
Important: You must upload the public key to your OCI user account. The setup process will display the public key content that you need to add.

Upload Public Key to OCI Console

  1. Log in to the OCI Console
  2. Navigate to Identity → Users
  3. Click on your username
  4. Under Resources, click API Keys
  5. Click Add API Key
  6. Select Paste Public Key
  7. Paste the content from ~/.oci/oci_api_key_public.pem
  8. Click Add

Configuration File Format

Your OCI configuration file (~/.oci/config on Linux/macOS or %USERPROFILE%\.oci\config on Windows) should look like this:

[DEFAULT]
user=ocid1.user.oc1..aaaaaaaaexampleuserocid
fingerprint=12:34:56:78:90:ab:cd:ef:12:34:56:78:90:ab:cd:ef
key_file=~/.oci/oci_api_key.pem
tenancy=ocid1.tenancy.oc1..aaaaaaaaexampletenancyocid
region=us-ashburn-1
Note: You can create multiple profiles in the configuration file by adding sections with different names (e.g., [PRODUCTION], [DEVELOPMENT]).

Test Your Configuration

Test your OCI CLI configuration by listing your compute instances:

oci compute instance list --compartment-id <your-compartment-ocid>

Or test with a simple command that does not require a compartment ID:

oci iam region list

If configured correctly, you should see a JSON response with a list of OCI regions.

Common OCI CLI Commands

  • oci compute instance list - List compute instances
  • oci os bucket list - List Object Storage buckets
  • oci network vcn list - List Virtual Cloud Networks
  • oci iam user list - List IAM users
  • oci iam compartment list - List compartments

For help with any command, use:

oci <service> <resource> <action> --help

Troubleshooting

Command not found

If you receive a command not found error, add OCI CLI to your PATH:

Windows:

Add %USERPROFILE%\bin to your PATH environment variable

macOS/Linux:

export PATH=$PATH:~/bin
# Add to ~/.bashrc or ~/.zshrc to make permanent

Authentication errors

If you encounter authentication errors, verify:

  • Your API key is properly uploaded to the OCI Console
  • The fingerprint in your config file matches the fingerprint shown in the OCI Console
  • Your private key file path is correct and the file has proper permissions (chmod 600 on Linux/macOS)
  • Your user OCID and tenancy OCID are correct

Permission errors

Ensure your OCI user has the necessary IAM policies to perform the requested actions.
